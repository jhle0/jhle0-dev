# 노션에서 블로그로 옮기는 작성 규칙

이 블로그는 노션에 정리한 문서를 Markdown으로 옮기기 쉽게 맞춰져 있습니다.  
특히 블로그 상세 문서는 노션처럼 문단, 토글, 체크리스트, 표, 수식, 이미지 캡션이 자연스럽게 보이도록 스타일링되어 있습니다.

## 가장 추천하는 흐름

1. 노션에서 문서를 먼저 정리
2. `/admin`에서 새 글 생성
3. 제목, 설명, 슬러그, 태그 입력
4. 노션 본문을 붙여넣고 블록을 CMS에서 정리
5. 필요한 부분은 `Callout`, `Toggle`, `Math Block`, `Figure` 블록으로 보강

## 글 작성 규칙

- 제목은 frontmatter의 `title`로 관리하므로 본문 맨 위에 다시 H1을 넣지 않는 편이 깔끔합니다.
- 설명(`description`)은 목록 카드와 OG 메타에 들어가므로 1~2문장으로 짧게 적는 것이 좋습니다.
- 태그는 자유 입력입니다. 새 태그를 넣으면 그대로 태그 페이지가 생깁니다.
- 시리즈 글은 `series`를 같은 값으로 맞추면 자동으로 연결됩니다.

## 노션 블록 대응 규칙

### 일반 문단

노션 본문 문단은 그대로 붙여넣어도 무난합니다.

### 제목

- 큰 섹션: `##`
- 하위 섹션: `###`

너무 깊은 단계의 제목은 피하는 편이 TOC가 더 깔끔합니다.

### 체크리스트

```md
- [ ] 해야 할 일
- [x] 완료한 일
```

### 접기 / 펼치기

CMS에서 `Toggle` 블록을 넣거나 아래 형식을 그대로 써도 됩니다.

```html
<details><summary>보조 설명 보기</summary>

숨겨둘 내용을 적습니다.

</details>
```

### 콜아웃

CMS에서 `Callout` 블록을 넣는 방식을 추천합니다.

출력 형식은 아래와 같습니다.

```html
<blockquote class="notion-callout">
  <p><strong>메모</strong></p>
  <p>강조하고 싶은 내용을 적습니다.</p>
</blockquote>
```

### 수식

인라인 수식:

```md
$E = mc^2$
```

블록 수식:

```md
$$
\int_0^1 x^2 dx = \frac{1}{3}
$$
```

### 이미지와 캡션

CMS의 `Figure` 블록을 쓰는 쪽이 가장 안정적입니다.

```html
<figure>
  <img src="/images/uploads/example.png" alt="설명" />
  <figcaption>이미지 캡션</figcaption>
</figure>
```

### 표

Markdown 표나 HTML 표 둘 다 괜찮습니다.  
복잡한 표는 HTML이 더 안정적입니다.

## 실전 팁

- 노션에서 복사해온 뒤 문단 간격만 먼저 정리해도 읽기 흐름이 크게 좋아집니다.
- 코드 블록은 언어를 꼭 지정하는 편이 좋습니다.
- 본문 첫 문단은 너무 길지 않게 쓰는 편이 목록 카드와 상세 헤더 모두에서 읽기 좋습니다.
- 이미지가 많으면 각 섹션 사이에 짧은 설명 문단을 넣어 흐름을 끊어주는 것이 좋습니다.

## CMS에서 이미 준비된 것

- 블로그 기본 본문 템플릿
- 노션형 프리뷰 레이아웃
- `Callout`, `Toggle`, `Math Block`, `Figure` 에디터 블록
- 자유 태그 입력
- 이미지 업로드 경로 자동 연결
